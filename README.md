# SMB Monitor

Платформа мониторинга ключевых показателей бизнеса, ролей, подтверждения данных, аналитики и интеграций.

Проект очищен от визуального тестового прототипа. Актуальная последовательность реализации хранится в `implementation_plan.md`.

Карта данных, типов аккаунтов и серверных границ хранится в `docs/data-architecture.md`.
Канонический визуальный стиль проекта хранится в `docs/visual-style.md`.

## Стек

- Vite
- React
- TypeScript
- CSS

## Локальный запуск

```bash
npm install
npm run dev -- --host 127.0.0.1
```

После запуска открой локальную ссылку, которую покажет Vite. Обычно это:

```text
http://127.0.0.1:5173/
```

## Как сейчас работает сервер

Сейчас в проекте нет отдельного backend-приложения и нет базы данных. Локальный сервер — это Vite dev/preview middleware в `vite.config.ts`.

Он обрабатывает:

- `GET /api/access/profile` — возвращает текущий серверный профиль доступа или пустой профиль;
- `POST /api/dev/access-session` — временно выбирает тип аккаунта для dev-режима;
- `DELETE /api/dev/access-session` — очищает выбранный dev-доступ.

Временная dev-сессия хранится в памяти процесса Vite и связана с браузером через HttpOnly cookie. После рестарта `npm run dev` выбранный тип аккаунта сбросится, и нужно будет снова нажать одну из трёх кнопок авторизации.

Реальные бизнес-данные сейчас нигде не сохраняются: БД ещё не подключена, worker-форма пока является UI/server-write placeholder. Поэтому при рестарте нечего потерять из production-данных. Постоянное хранение форм и статистики появится отдельным backend/storage шагом.
